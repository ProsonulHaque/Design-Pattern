Overview:

- Structural desgin pattern.

- Frontface interface.

- Simpler interface for complex module/subsystem.

- Basically use complex system/subsystem/module.

- Basically use a single wrapper class contains set of members consist of different classes.

- Implement a single interface that delegate request to subsystems.

Benifits:

- Improve readability

- Hide complixity

- Loosely couple code

- Easier to use.

- Minimized dependencies from subsystems/modules.

- May add additional functionality before/after forwading request.
