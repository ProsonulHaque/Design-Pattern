Overview:

- Structural design pattern

- Main goal is to seperate the abstraction from implementation so that they can vary independently.

- 2 parts 1) Abstraction 2) Implementation.

- Client code can use only abstraction.

- State, stratigy, Bridge are similar pattern.

Befinits:

- Without clientcode change, possible to change implementation details.

- Increase loose coupling.

- Separating low level logic implementation from high level defination.
